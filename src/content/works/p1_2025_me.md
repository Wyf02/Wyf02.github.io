## Overview

Personal portfolio built entirely from scratch — no UI library, no CSS framework, no animation library. Every visual effect, layout system, and interaction is hand-implemented. The goal was to demonstrate full-stack frontend capability: from webpack configuration and build pipeline design, down to raw Canvas API rendering and browser performance optimization.

---

## Engineering Decisions

### Why Vue CLI 5 + Options API

Deliberately chose Options API over Composition API for a solo project at this scale — the lifecycle hooks and computed/methods separation maps cleanly to the mental model of "what does this component do," and avoids the overhead of `setup()` boilerplate when the component count is manageable. Vue Router 4 with **hash history** was chosen specifically for static hosting compatibility on GitHub Pages, which has no server-side routing fallback.

### Lazy-loaded Route Chunks

All non-home routes are dynamically imported with webpack magic comments:

```js
component: () => import(/* webpackChunkName: "work-detail" */ '../views/WorkDetailView.vue')
```

This splits the bundle so the initial load only ships the home page JS, keeping Time-to-Interactive low for first-time visitors.

## Canvas Rendering Engine

Neither canvas animation uses any external library — both are written against the raw Canvas 2D API with full control over every pixel.

### Home: Real-Time Animated Wave System

Three sine waves rendered at 60fps via a `requestAnimationFrame` loop. Each wave is an independent layer with its own amplitude, wavelength, speed, and alpha:

```js
waves: [
  { amp: 12, len: 0.012, speed: 0.015, color: 'rgba(0,150,255,0.25)' },
  { amp: 20, len: 0.008, speed: 0.010, color: 'rgba(0,120,255,0.20)' },
  { amp: 34, len: 0.005, speed: 0.007, color: 'rgba(0, 90,255,0.15)' },
]
```

A `t` counter increments each frame; each wave's phase offset is `t × speed`, so layers animate at different rates and produce emergent interference patterns. Floating particles ride along each wave's surface — their x positions are computed with modular arithmetic so they cycle seamlessly without any state array:

```js
const px = ((p * 97 + this.t * (0.4 + i * 0.15)) % this.width)
const py = baseline + Math.sin(px * len + this.t * speed) * amp
```

**DPI awareness**: canvas is sized to `window.innerWidth × devicePixelRatio` and the context transform is set via `ctx.setTransform(dpi, 0, 0, dpi, 0, 0)`, so rendering is crisp on Retina/HiDPI displays without any CSS blur.

**Memory management**: the RAF id is stored in `this.rafId` and cancelled in `beforeUnmount`, and the resize listener is registered with `{ passive: true }` to avoid blocking the main thread's scroll handling.

### Works / Misc: Procedural Mountain Silhouettes

Drawn once on mount, redrawn only on viewport resize — no animation loop needed. Each mountain ridge is a sum of two sine functions at different frequencies, approximating a natural skyline:

```js
y = base - (sin(x × 0.004) + sin(x × 0.011 + 2)) × 0.5 × amp - random() × amp × 0.08
```

The small `random()` term adds subtle roughness on each redraw. Three layers with increasing foreground color brightness create depth via atmospheric perspective. The same `drawMountains` function is shared across three views (WorksView, WorkDetailView, MiscView) — each instantiates its own canvas with identical logic.

## Markdown Content Pipeline

Content for each work detail page lives in `.md` files. The full pipeline:

**Step 1 — webpack `asset/source` rule** in `vue.config.js`:
```js
config.module.rule('md').test(/\.md$/).type('asset/source')
```
This tells webpack to emit each `.md` file's raw text as a JS string module — no external loader package needed.

**Step 2 — Static imports** in `worksContent.js`:
```js
import r1 from './works/r1_pvis2025_art.md'
```
webpack resolves these at build time, so all content is bundled and available instantly — no async fetch, no loading state for content.

**Step 3 — Single-pass TOC injection**: rather than two separate passes (one to parse headings, one to re-render), a single `String.replace` with a capture-group regex simultaneously injects `id` attributes into the HTML *and* builds the `toc[]` array:

```js
html.replace(/<(h[23])([^>]*)>([\s\S]*?)<\/h[23]>/gi,
  (match, tag, attrs, inner) => {
    const text = inner.replace(/<[^>]+>/g, '').trim()
    // slug generation with duplicate handling
    toc.push({ id: slug, text, level: parseInt(tag[1]) })
    return `<${tag}${attrs} id="${slug}">${inner}</${tag}>`
  }
)
```

Slug deduplication tracks a `slugCount` map to handle repeated headings gracefully.

## Scroll-Tracked Table of Contents

The sidebar TOC stays in sync with reading position using `IntersectionObserver` — no scroll event listener, no `getBoundingClientRect` polling on every scroll frame.

```js
new IntersectionObserver(entries => {
  entries.forEach(e => { if (e.isIntersecting) this.activeId = e.target.id })
}, { rootMargin: '0px 0px -80% 0px', threshold: 0 })
```

`rootMargin: '0px 0px -80% 0px'` means the observer's detection window is the top 20% of the viewport — a heading fires `isIntersecting` exactly when it scrolls into the reader's active attention zone.

**Click-to-scroll with navbar compensation**: `scrollIntoView` doesn't account for the fixed navbar. Manual scroll calculation:

```js
scrollTo(id) {
  const top = el.getBoundingClientRect().top + window.scrollY - window.innerHeight * 0.12
  window.scrollTo({ top, behavior: 'smooth' })
}
```

`getBoundingClientRect().top` is viewport-relative; adding `window.scrollY` converts it to document-absolute; subtracting `12vh` clears the navbar with breathing room.

The observer is disconnected in `beforeUnmount` — missing this is a common source of memory leaks in SPA navigation.

## CSS Architecture

### Scoped Styles + `:deep()` for v-html Content

Markdown is injected via `v-html`, which renders outside Vue's scoped attribute scope. Rather than using a global stylesheet (which would pollute all pages), all markdown typography styles use Vue 3's `:deep()` combinator:

```css
:deep(.content) h2 { ... }
:deep(.content) blockquote { ... }
```

This preserves style encapsulation while targeting dynamically injected DOM.

### 3D Flip Animation — Pure CSS

The cycling tagline uses `rotateX` 3D transforms coordinated with Vue's `<transition>` component. No JS, no animation library:

```css
.flip-leave-active { animation: lay-down 520ms cubic-bezier(.2,.8,.2,1); }
.flip-enter-active { animation: flip-up  520ms cubic-bezier(.2,.8,.2,1); }

@keyframes lay-down {
  to { opacity: 0; transform: rotateX(90deg) translateY(4px); }
}
@keyframes flip-up {
  from { opacity: 0; transform: rotateX(-90deg) translateY(-4px); }
}
```

`perspective: 800px` on the parent establishes the 3D rendering context. The `cubic-bezier(.2,.8,.2,1)` easing gives a snappy deceleration that reads as physical weight.

### Responsive Typography with `clamp()`

Font sizes scale fluidly between viewport widths without stepped media query breakpoints:

```css
font-size: clamp(40px, 8vw, 80px);
```

This eliminates the jarring snap between breakpoints and works across any screen size.

### Notion-Style Full-Bleed Banner

The cover image banner needs to extend behind the fixed navbar (which sits above the `padding-top: 10vh` page container). Solution: negative margin to pull the banner up exactly that amount:

```css
.banner { margin-top: -10vh; }
```

Cleaner than overriding the global `.page` padding or using `position: absolute`.

## Layout Systems

### Works Sidebar: Sticky Metadata + TOC

Left column is `position: sticky; top: 32px` — it scrolls with the page until it hits the viewport edge, then pins. The TOC appended to the bottom of the same sticky column means the navigation is always visible regardless of how far down the user has read.

### Masonry Photo Gallery

CSS `column-count: 3` masonry with zero JavaScript layout calculation. Images render via `<img width="100%" height="auto">` so each card's height is determined by the photo's natural aspect ratio — producing organic column height variation without any forced fixed heights or layout recalculation.

### Faceted Filter (Works Page)

Multi-dimensional client-side filtering across Role × Tags × Language × Tech × Status, combined with free-text search over all string fields. Implemented as a single Vue `computed` property — reactive, zero-latency, no debouncing needed for in-memory filtering.
